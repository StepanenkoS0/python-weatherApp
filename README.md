# python-weatherApp
# Прогноз опадів на основі даних Open-Meteo

## Опис проєкту

Даний проєкт є прототипом міні-сервісу прогнозування опадів.
Програма отримує історичні метеодані з сервісу Open-Meteo через HTTP-запит, формує датасет, навчає модель машинного навчання та дозволяє зробити прогноз опадів.

Інтерфейс реалізовано у вигляді веб-застосунку на Streamlit, де користувач може отримати дані, навчити модель та виконати прогноз одним натисканням кнопки.

---

# Дані, що використовуються

Дані отримуються з API сервісу
Open-Meteo

Запит виконується до архівного API:

```
https://archive-api.open-meteo.com/v1/archive
```

Використовуються **щоденні метеодані (daily)**.

Основні поля датасету:

| Поле               | Опис                             |
| ------------------ | -------------------------------- |
| time               | дата                             |
| precipitation_sum  | сумарна кількість опадів за день |
| rain_sum           | кількість дощу за день           |
| temperature_2m_max | максимальна температура за день  |
| temperature_2m_min | мінімальна температура за день   |
| wind_speed_10m_max | максимальна швидкість вітру      |

Цільова змінна формується на основі precipitation_sum:

```
precipitation_sum = 0  → 0 (опадів немає)
precipitation_sum > 0 → 1 (опади є)
```

---

# Використана модель

Для задачі класифікації використовується модель:

**Logistic Regression**

Бібліотека:

```
scikit-learn
```

Ознаки (X):

```
temperature_2m_max
temperature_2m_min
wind_speed_10m_max
```

Цільова змінна (y):

```
rain_label
```

Модель навчається на частині даних та перевіряється на тестовій вибірці.

Розбиття датасету:

```
train/test = 80% / 20%
```

Для оцінки якості використовуються метрики:

* accuracy
* precision
* recall
* f1-score

## Опис файлів

### app.py

Основний файл Streamlit-застосунку.

Виконує такі функції:

1. приймає координати та період дат;
2. робить HTTP-запит до Open-Meteo;
3. отримує та відображає датасет;
4. зберігає дані у CSV;
5. викликає навчання моделі;
6. відображає метрики якості;
7. виконує прогноз опадів.

---

### model.py

Файл з логікою машинного навчання.

Містить дві функції:

**train_model()**

* формує цільову змінну
* створює X та y
* ділить дані на train/test
* навчає модель Logistic Regression
* обчислює метрики

**predict_rain()**

* приймає модель та нові дані
* виконує прогноз
* повертає клас прогнозу та ймовірність.

# Послідовність роботи програми

1. Користувач задає координати та період дат.
2. Програма виконує HTTP-запит до Open-Meteo.
3. Отримані щоденні метеодані перетворюються у DataFrame.
4. Дані зберігаються у файл weather_daily.csv.
5. При натисканні кнопки "Навчити модель":

   * формується цільова змінна;
   * дані діляться на train/test;
   * навчається модель Logistic Regression;
   * обчислюються метрики якості.
6. При натисканні кнопки "Зробити прогноз":

   * беруться останні доступні дані;
   * модель визначає наявність опадів;
   * виводиться результат та ймовірність.
