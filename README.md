# python-weatherApp
# ML Прогноз опадів (Open-Meteo)

Цей проєкт реалізує міні-сервіс прогнозування опадів на основі історичних метеоданих та машинного навчання. Застосунок дозволяє користувачу вводити місто та період дат, отримувати історичні дані, навчати модель та робити прогноз на останній день вибраного періоду.

---

## 1. Використані дані

- Джерело: [Open-Meteo](https://open-meteo.com/) через HTTP-запити (безкоштовно).  
- Тип даних: **щоденні** (`daily`), а не погодинні.  
- Обов’язкові поля:
  - `precipitation_sum` — сумарні опади за день (включає дощ, сніг тощо)
  - `rain_sum` — сумарна кількість дощу за день  
- Додаткові ознаки:
  - `temperature_2m_max` — максимальна температура
  - `temperature_2m_min` — мінімальна температура
  - `windspeed_10m_max` — максимальна швидкість вітру  
- Цільова змінна (label):
  - `rain_label`:
    - 0 — опадів немає
    - 1 — опади є  

Дані зберігаються у CSV-файл `weather_daily.csv` після отримання історичних записів.

---

## 2. Використана модель

- Модель класифікації: **Random Forest Classifier** з бібліотеки `scikit-learn`.  
- Ознаки для навчання: `rain_sum`, `temperature_2m_max`, `temperature_2m_min`, `windspeed_10m_max`.  
- Мета: передбачити ймовірність настання опадів на конкретний день (`rain_label`).  
- Метрики оцінки:
  - **Точність (accuracy)** на тестовій вибірці
  - **Важливість факторів** (feature importance) для розуміння впливу кожної ознаки

   - Відображення таблиці історичних даних та графіку сумарних опадів.


   - Опис проєкту, використані дані, модель та структура файлів.

## 4. Послідовність дій

1. Користувач вводить місто, дату початку та дату кінця.
2. Натискає **"Отримати історичні дані"**:
   - Дані завантажуються з Open-Meteo.
   - Таблиця і графік опадів відображаються на екрані.
   - CSV-файл `weather_daily.csv` створюється локально.
3. Натискає **"Навчити модель"**:
   - Модель Random Forest навчається на ознаках.
   - Відображається точність та важливість факторів.
4. Натискає **"Прогноз"**:
   - Модель передбачає настання опадів на останній день періоду.
   - Виводиться прогноз та ймовірність опадів.

---

## 5. Примітки

- Проєкт повністю працює через веб-застосунок Streamlit (`weatherApp.py`).
- Користувач може вводити будь-яке місто, дата початку та кінця повинні бути у форматі `YYYY-MM-DD`.
- Прогноз формується **на останній день вибраного періоду**, оскільки весь період вже доступний для моделі.
